Four deliberately separate layers

The map never shows a generic “actual ET forecast.” Each layer has its own meaning, source path, and eligible timestamps, named verbatim from the product contract.

LayerMeaningTime semantics
eto_mmWeather-driven ASCE short-reference ET forecastFuture daily forecast from ensemble weather members
potential_et_c_mmKc × ETo under ample-water conditionsFuture daily, conditional on the crop-coefficient assumption
eta_analysis_mmLatest observed-date satellite ET analysisHistorical observation at source lag; never a future forecast
eta_well_watered_mmConditional scenario: crop water not limitingFuture daily scenario, not a measurement
eta_no_irrigation_mmConditional scenario: no irrigation after issue timeFuture daily scenario, not a measurement

Phase 2 evidence — field-withheld daily ET

85 flux-tower stations, field-withheld split. OpenET assimilation reduced daily actual-ET MAE by 43.4% against the best OpenET-free baseline (Δ 0.658 mm/day, 95% CI [0.399, 0.911]).

Reproduce

From the repository root
# build an immutable outlook run from the example fixtures, # then render the same map candidate served on this site python3 -m pip install -e . mlet build-outlook --weather examples/outlook/weather_members.jsonl \ --state examples/outlook/state.jsonl --crop examples/outlook/crop_grid.jsonl --out runs mlet publish-outlook --run runs/RUN_ID
